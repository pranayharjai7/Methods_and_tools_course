🔭 PCA — Reducing 5D to 1D

Principal Component Analysis: find the direction that captures maximum variance

The Problem: 5 Correlated Features

Look at the exam data. The 5 columns are highly correlated — they all move together. This makes sense when you look at the ratios:

The exam data: x0 through x4 are nearly linearly dependent

For a sample row: x0=21.99, x1=67.44, x2=47.13, x3=180.04, x4=112.61

Ratios: x1/x0 ≈ 3.07, x2/x0 ≈ 2.14, x3/x0 ≈ 8.19, x4/x0 ≈ 5.12

These ratios are roughly constant across all rows. This means the 5 features essentially lie on a 1D line in 5D space. PCA will find that line.

What is PCA? (The Intuition)

Imagine 100 data points in 3D space — but they all lie roughly on a flat pancake (2D plane). You can describe each point much more compactly using just its 2D position on that pancake.

PCA finds the axes of that pancake — the directions where the data varies the most. These are called Principal Components.

The Mathematics of PCA

1Normalise each feature to zero mean and unit variance.
X_norm = (X - mean) / std
Why? Without this, features with large values dominate. In the exam data, x3 is 8× larger than x0 — without normalisation, x3 would appear most important just because of its scale.
2Compute the covariance matrix: Σ = (1/m) · Xᵀ · X
Why? This (n×n) matrix captures how each pair of features varies together. Off-diagonal = correlation.
3Eigendecomposition: eigenvalues λ, eigenvectors U = np.linalg.eig(Σ)
Why? The eigenvectors are the principal directions. The eigenvalue = variance in that direction.
4Sort by eigenvalue (largest first). The first eigenvector is PC1 — the direction of maximum variance.
5Project: x₁ = X_norm @ U[:, :1]
This gives a single number for each data point — its coordinate along PC1.

Explained Variance: How Much Information is Preserved?

The explained variance ratio tells you what fraction of the total variance is captured by k components:

explained = sum(eigenvalues[:k]) / sum(all eigenvalues)

For the exam data, because the features are nearly perfectly correlated, PC1 alone explains ~99%+ of the variance. Reducing to 1D loses almost nothing.

The Exam Code (Problem 1b)

✅ Full PCA from scratch — no sklearn

# Step 1: Normalise
mu    = X.mean(axis=0)          # shape (5,) — mean of each feature
sigma = X.std(axis=0)           # shape (5,) — std of each feature
X_norm = (X - mu) / sigma       # shape (m, 5) — each feature: mean=0, std=1

# Step 2: Covariance matrix
m = X_norm.shape[0]
Sigma = (1/m) * X_norm.T @ X_norm   # shape (5, 5)

# Step 3: Eigendecomposition
eigenvalues, eigenvectors = np.linalg.eig(Sigma)

# Step 4: Sort descending by eigenvalue
order = np.argsort(eigenvalues)[::-1]
eigenvalues  = eigenvalues[order].real
eigenvectors = eigenvectors[:, order].real

# Step 5: Project onto PC1
U1 = eigenvectors[:, :1]        # shape (5, 1) — first principal component
x1 = X_norm @ U1                # shape (m, 1) — 1D representation
x1 = x1.flatten()               # shape (m,)

explained = eigenvalues[0] / eigenvalues.sum()
print(f"PC1 explains {explained:.1%} of variance")  # Algorithm: PCA

⚠️ The exam says "Specify the name of the algorithm used" — the answer is PCA (Principal Component Analysis)

Applying to Test Data

⚠️ Critical: Use training statistics on test data

When you normalise test data and project it, you must use the mean (mu) and std (sigma) from the training set, and the eigenvectors from the training covariance matrix.

# Normalise test data using TRAINING mu and sigma
X_test_norm = (X_test - mu) / sigma    # same mu, sigma from training!

# Project using TRAINING eigenvectors
x1_test = (X_test_norm @ U1).flatten()

If you computed new mu/sigma from test data, the 1D coordinates would be on a different scale — your predictions would be wrong.

What PCA Actually Does to the Exam Data

Since x0 through x4 are nearly linearly dependent (x1 ≈ 3·x0, x3 ≈ 8·x0, etc.), PCA essentially finds the overall magnitude of the data point. All five features go up or down together, and PC1 captures this common factor.
